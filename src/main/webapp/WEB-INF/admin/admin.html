git学习系列（10）--->>bug分支

1、master分支最好是稳定的，所有的开发建议在新分支dev上，稳定后merge到master中。

2、会有一种情况，就是你在dev上开发，但是同时有可能需要修改一个master上的bug，这个时候需要选保存dev的工作空间，然后切换到master上新建一个bug分支。

--------------------------------------------------------------

git stash  // 保存工作空间

git checkout master

git checkout -b bug_001

一系列的add/commit后

切换到master分支  git branch master

git merge bug_001

git checkout dev

// 此时通过git status查看工作空间为空

git stash list // 查看保存的工作空间

git stash pop // 恢复工作空间

也可以指定某个工作空间恢复 :git stash apply stash@{0}

git学习系列（9）--->>合并分支冲突问题解决

1、当一个文件在两个分支都修改后，提交的时候就会出现冲突

2、这种情况下只能通过人工合并冲突文件来解决冲突。

3、最后删除分支

---------------------------------------------------------------

git checkout -b dev

// 修改test.txt文件 并提交

git branch master  // 切换到master分支

// 修改test.txt文件 并提交

// 在master分支上合并dev

git merge dev // 发现冲突了

只能vi test.txt文件，修改后add/commit，这才解决

最后删除分支dev，git  branch  -d dev

---------------------------------------------------------------

git学习系列（8）--->>分支使用


1、查看分支：

git branch

2、创建分支：

$ git checkout -b dev   // -b表示创建并切换

另外一种方式：git branch dev | git checkout dev

3、这个时候查看分支：

$ git branch

* dev  // 当前分支带*号
master

4、切换分支：

git checkout master

5、合并分支：

git merge dev  // 合并指定分支到当前分支

6、删除分支：

git branch -d dev
